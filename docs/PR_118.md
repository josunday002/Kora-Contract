### Implementation Details

#### Role Management State Machine
Valid role transitions:
```
None ←→ Operator (grant/revoke)
None ←→ Verifier (grant/revoke)
None ←→ Admin (transfer_admin only)
Operator ←→ Verifier (can overwrite)
```

Constraints:
- Cannot grant Admin (use transfer_admin)
- Cannot grant None (use revoke_role)
- Cannot grant role to current admin
- Cannot revoke admin's own role

#### Admin Transfer Protocol
Transfer process:
1. Current admin authenticates
2. Validate new admin address is valid
3. Validate new admin doesn't hold conflicting role
4. Update admin address
5. Set new admin's role to Admin
6. Clear old admin's role
7. Emit event

Tests verify:
- New admin gains all privileges
- Old admin loses all privileges
- Process is atomic

#### Pause State Consistency
Pause semantics:
- Can only pause when not paused (AlreadyPaused error)
- Can only unpause when paused (NotPaused error)
- Only admin can pause/unpause
- Used by marketplace and invoice_nft contracts

Tests verify:
- State transitions are validated
- Double operations fail with specific errors
- Admin can pause/unpause multiple times
